Design Questions

  1. Explain how design and development intersect. What role do you see a developer playing in the design process?

    I believe that design and development are completely connected. If the designer and the developer are not the same person, then they must be in constant communication and both agree on what they are working to achieve. The designer may come up with they layout and the look and feel of the website, but the developer has to be able to implement the design. I believe that the developer's job is to implement the design as accurately as possible but also communicate any limitations that may arise.

  2. Describe how grids are useful, and how you generally go about using them, or implementing them.

    Grids are useful for easily laying out content on a page. They are also great for building pages that are responsive. Generally I use the grid systems that come with front-end frameworks like Twitter Bootstrap. I have also had some exposure to creating a grid system from scratch from teamtreehouse.com. Currently, I am interested in learning how to use flexbox to see if it would be an easier tool for page layout.

  3. Talk a little bit about pattern libraries and visual systems. What are they and how are they useful? Have you used them before?

    To make development easier and more consistant pattern libraries are used to detail how elements should look and respond. I have not had experience with either pattern librarys or visual systems but I am always ready to learn a new way to make my web development projects easier to create and maintain.

  7. Explain how HTML, CSS, and JS interact - what is each layer responsible for?

    HTML provides the structure for the content of a web page. CSS provides the styling, the look and feel of the content on the page. CSS is also used for the layout of the content. JavaScript provides the behavior for elements on the webpage and handles user interaction.

  8. Progressive enhancement or graceful degredation? Choose your side.

    Progressive enhancement allows for a web applications base functionality to be built first and then incremnetally upgraded. This seems easier and more efficient than trying to build the final version of the application and then figuriing out what functionality needs to be removed for less modern browsers.
